# Job Papers Optimize

여러 회사와 직무에 지원하는 한 명의 구직자를 위한 로컬 AI 작업환경이다. 이력서, 포트폴리오, 자기소개서와 면접 준비를 회사별로 맞춤화하되, 실제 경력과 원본 문서의 형식을 지키는 데 초점을 둔다.

Claude Code, Codex, Antigravity처럼 내 컴퓨터의 폴더를 열어 대화할 수 있는 AI 에이전트에서 사용한다. 터미널 명령, 코딩 지식, 별도 개발 도구 설치는 필요하지 않다.

## 이런 사람에게 맞습니다

- 여러 회사에 지원하지만, 자신의 경력과 프로젝트를 매번 처음부터 설명하고 싶지 않은 사람
- 기존 이력서나 포트폴리오의 양식은 유지하면서 회사와 직무에 맞게 표현과 강조를 바꾸고 싶은 사람
- 채용공고, 회사 조사, 지원 전략, 면접 준비를 한 흐름으로 정리하고 싶은 사람
- 사실에 없는 경력이나 성과를 만들지 않는 방식으로 지원 자료를 다듬고 싶은 사람

여러 지원자를 관리하는 채용 컨설팅 업무나 유료 서비스 운영을 위한 도구는 아니다.

## 할 수 있는 일

- 이력서, 포트폴리오, 자기소개서, 프로젝트 자료를 바탕으로 재사용 가능한 지원자 프로필 만들기
- 회사·제품·직무를 순서대로 조사하고, 확인된 정보와 확인되지 않은 정보를 구분하기
- 직무 요구사항, 인재상, 문화·업무 방식, 제품 맥락을 내 경력의 검증된 근거와 연결해 지원 전략 세우기
- 원본 문서의 양식과 구조를 지키며 어휘, 강조점, 분량, 글머리표, 경력·프로젝트의 우선순위 조정하기
- 실무·임원·문화 면접 질문, 답변 구조, 역질문 준비하기
- 새 경력이나 프로젝트가 생겼을 때 기존 프로필의 변경안만 만들고, 승인 후 반영하기

원본에 없는 경력, 성과, 수치, 도구 경험은 만들지 않는다. 조사로 확인되지 않은 회사 분위기나 문화도 사실처럼 쓰지 않는다.

## 시작하기

### 1. GitHub에서 받기

1. [프로젝트 페이지](https://github.com/houarnu166/job-papers-optimize)를 연다.
2. 초록색 **Code** 버튼을 누른다.
3. **Download ZIP**을 눌러 내려받는다.
4. 내려받은 ZIP 파일의 압축을 푼다.

압축을 풀면 `job-papers-optimize` 또는 비슷한 이름의 폴더가 생긴다. 그 폴더 안에 `AGENTS.md`, `inputs`, `orders`, `outputs`, `skills`가 보이면 올바른 폴더다.

### 2. AI 에이전트에서 폴더 열기

Claude Code, Codex 또는 Antigravity를 연 뒤 **폴더 열기**, **프로젝트 열기**, **워크스페이스 열기**와 비슷한 메뉴를 선택한다. 방금 압축을 푼 `job-papers-optimize` 폴더 자체를 연다.

그 폴더에서 새 대화나 작업을 시작하면 된다. 주문서나 설정 파일을 직접 만들 필요는 없다.

### 3. 첫 대화 시작하기

처음에는 아래 문장을 그대로 보내도 된다.

> 이 폴더의 AGENTS.md와 로컬 skills를 먼저 읽고 따라줘. 처음 사용하는 환경이니 내 지원자 프로필부터 만들어줘.

개인 자료는 대화에 직접 첨부하거나 `inputs/private/sources/` 폴더에 넣는다. 원본 파일은 에이전트가 수정하지 않는다.

## 이렇게 대화할 수 있습니다

처음 프로필을 만들 때:

> 내 이력서와 포트폴리오를 첨부했어. 앞으로 여러 회사 지원에 재사용할 지원자 프로필을 만들어줘.

특정 회사에 지원할 때:

> 이번에는 아크랩 Product Designer에 지원하려고 해. 채용공고와 현재 이력서·포트폴리오를 첨부했어. 기존 프로필을 사용해서 회사와 직무를 조사하고, 두 문서의 양식은 유지한 채 Markdown으로 최적화해줘. 면접 준비도 해줘.

이력서만 빠르게 다듬을 때:

> 이 채용공고에 맞춰 현재 이력서만 최적화해줘. 원본 형식은 유지하고, 내 경험 중 무엇을 앞세울지 전략부터 정리해줘.

프로필에 새 경험을 추가할 때:

> 새 프로젝트 자료를 추가했어. 기존 프로필에서 달라질 내용만 제안해줘. 아직 기준 프로필에는 반영하지 마.

에이전트는 필요한 경우에만 회사, 직무, 채용공고, 지원 단계, 원하는 결과물을 짧게 확인한다.

## 한 회사 지원은 이렇게 진행됩니다

1. 승인된 지원자 프로필과 실제 제출 원본을 확인한다.
2. 회사, 제품, 직무, 인재상과 업무 맥락을 조사한다.
3. 회사가 찾는 신호와 내 경력의 검증된 근거를 매핑해 지원 전략을 만든다.
4. 그 전략에 따라 원본 양식 안에서 표현, 강조, 분량, 글머리표와 경력·프로젝트의 우선순위를 조정한다.
5. 필요한 범위의 면접 준비를 만든다.
6. 사실성, 원본 형식 보존, 이미지·표, 제출 준비 상태를 검수한다.

같은 섹션 안의 글머리표나 기존 경력·프로젝트 단위는 전략적으로 순서를 바꿀 수 있다. 반면 섹션을 새로 만들거나 없애기, 문단·블록을 합치거나 나누기, 최상위 섹션을 옮기기, 레이아웃을 새로 설계하기는 사용자 승인 없이는 적용하지 않는다.

## 입력과 결과물

- 입력: 현재 AI 환경이 읽을 수 있는 PDF, DOCX, Markdown, HTML, JSON, TXT
- 기본 결과물: Markdown
- 선택 결과물: 현재 AI 환경이 지원하는 경우 DOCX 또는 HTML
- PDF: 읽기 전용 입력 자료. PDF 결과물은 만들지 않음

PDF나 DOCX를 읽을 수 없는 환경이라면 프로그램 설치를 요구하지 않는다. 필요한 목적에 따라 PDF, Markdown, TXT처럼 읽을 수 있는 대체 형식을 요청한다. DOCX 또는 HTML 내보내기를 지원하지 않으면 승인된 Markdown을 정식 결과물로 제공한다.

## 반복해서 사용할 때

프로필을 한 번 승인하면, 다음 회사에 지원할 때는 경력 사실을 다시 처음부터 정리하지 않는다. 회사와 직무가 바뀔 때마다 해당 회사의 요구와 내 강점을 새로 매칭하고, 실제로 제출할 원본 문서만 읽어 맞춤화한다.

새 경력이나 프로젝트가 생겼을 때만 프로필 갱신을 요청한다. 갱신안은 먼저 제안으로 만들며, 사용자가 승인하기 전까지 기준 프로필은 바뀌지 않는다.

## 개인정보

개인 원본은 `inputs/private/sources/`에, 승인된 지원자 프로필은 `inputs/profile/`에, 회사별 결과물은 `outputs/`에 보관한다. 이 경로의 실제 개인 자료와 결과물은 공유용 프로젝트에 포함하지 않도록 구성되어 있다.

개인정보가 포함된 자료를 어느 AI 서비스에 첨부할지는 해당 서비스의 개인정보 정책과 사용 환경을 확인한 뒤 결정한다.

개인 연구·학습·실험과 자신의 구직 준비에 사용할 수 있다. 이력서, 포트폴리오, 자기소개서, 지원서, 면접 자료를 만들고 채용 과정에 제출하는 개인 사용도 허용된다.

다음과 같은 용도는 허용되지 않는다.

- 프로젝트나 수정본을 판매, 재판매, 임대 또는 유료 배포하는 일
- 이 프로젝트를 이용해 타인에게 유료 이력서 작성, 취업 컨설팅, 코칭, 채용 대행 또는 교육을 제공하는 일
- 유료 제품, 서비스, 플랫폼, 사내 상업 서비스 또는 수익화된 업무 흐름에 포함하는 일

사용자가 제공한 이력서, 포트폴리오, 경력 정보와 개인정보의 권리는 사용자에게 있다. 개인 맞춤 결과물은 자신의 구직 목적으로 사용할 수 있다. 자세한 조건은 [LICENSE](LICENSE)를 확인한다.
