# Interview Preparation Report Contract

면접 자료의 분량과 형식이 실행할 때마다 달라지지 않도록 하는 고정 계약이다. 직무에 맞게 카테고리 이름과 질문 내용을 바꾸되 문서 구조와 질문별 필드는 유지한다.

## Scope

- `full`: 실무 12문항 이상, 임원·문화 12문항 이상, 포트폴리오·경력 검증, 공통·위험 질문, 역질문, 체크리스트
- `practical`: 실무 4개 카테고리 x 3문항 이상, 공통·위험 질문, 역질문, 체크리스트
- `executive`: 임원·문화 4개 카테고리 x 3문항 이상, 공통·위험 질문, 역질문, 체크리스트
- `portfolio`: 주요 프로젝트 또는 경력 사례별 3문항 이상, 소개·위험 질문, 역질문, 체크리스트
- `compact`: 사용자가 명시적으로 간결한 자료를 요청한 경우에만 핵심 8문항 이상. 질문별 필드는 유지한다.

면접 유형을 알 수 없고 사용자가 전체 준비를 요청하면 `full`을 기본값으로 사용한다.

## Interview Plan

`interview/interview-plan.md` 목차:

1. 면접 맥락과 scope
2. 회사·직무 평가 축
3. 지원자 한 문장 포지셔닝
4. 핵심 강점 3개와 evidence
5. 예상 우려 3개 이상과 대응 원칙
6. 제출 문서에서 면접관이 확인할 지점
7. 사용할 story bank와 중복 방지 계획
8. 질문 카테고리와 목표 문항 수
9. 사용자 확인이 필요한 개인 사실
10. 완료 체크리스트

## Final Report

`interview/interview-prep.md`는 다음 순서를 따른다.

```markdown
# [회사명] [직무명] 면접 예상 질문 및 답변

- 준비 기준일:
- 면접 범위:
- 답변 목표 시간:
- 사용한 제출 문서:
- 사용한 프로필 버전:

## 0. 핵심 면접 전략

### 0.1 한 문장 포지셔닝
### 0.2 30초 소개
### 0.3 60초 자기소개
### 0.4 핵심 어필 포인트 3개
### 0.5 예상 우려와 대응 원칙

## I. 실무 면접 예상 질문

### 주제 1. 핵심 직무 프로세스와 방법론

### 주제 4. 문제 해결·우선순위·실행

## II. 임원·문화 면접 예상 질문

### 주제 1. 비즈니스 이해와 전략적 사고
### 주제 2. 조직 문화와 가치관
### 주제 3. 리더십과 성장 가능성
### 주제 4. 산업 변화와 미래 관점

## III. 포트폴리오·경력 사례 검증

### 사례별 핵심 설명
### 사례별 심층 질문
### 개인 기여와 팀 기여 구분
### 실패·트레이드오프·회고 질문

## IV. 공통·위험 질문

### 실무자에게 할 질문
```

Scope에서 제외된 I 또는 II는 삭제할 수 있다. 이 경우 상단 metadata와 `interview-plan.md`에 제외 근거가 있어야 한다. 0, IV, V, VI은 모든 scope에 유지한다.

## Question Unit

각 질문은 다음 형식을 그대로 사용한다.

```markdown
#### 질문 [번호]. [실제 질문]

**질문 의도와 평가 기준**
- [면접관이 확인하려는 것]

**답변 전략**
- [어떤 사례와 순서로 말할지]

**암기용 핵심 키워드**
- Situation/Context:
- Task/Challenge:
- Action/Reasoning:
- Result/Learning:

**말하기용 모범 답변**
[60~120초 분량의 자연스러운 답변]

**근거와 확인 상태**
- Evidence: [EV-####]
- Status: verified | 사용자 답변 필요

**예상 꼬리질문**
1. [질문]
2. [질문]

**주의할 함정**
- [과장, 회사 추정, 역할 혼동, 수치 위험]
```

STAR가 맞지 않는 질문은 키워드를 `Position / Reason / Example / Application`처럼 바꿀 수 있다. 8개 필드 자체는 유지한다.

## Portfolio Deep-Dive

선택된 각 프로젝트·사례에는 다음 관점의 질문을 포함한다.

- 왜 그 문제를 선택했고 성공을 어떻게 정의했는가
- 본인이 직접 결정하고 수행한 범위는 무엇인가
- 근거가 부족하거나 첫 가설이 틀렸을 때 무엇을 바꿨는가
- 어떤 트레이드오프와 제약이 있었는가
- 결과 수치를 어떻게 측정했고 다시 한다면 무엇을 바꿀 것인가

모든 질문을 모든 사례에 반복할 필요는 없지만 사례당 3개 이상이어야 한다.

## Reverse Questions

- 최소 5개를 준비한다.
- 각 질문에 `대상`, `질문 의도`, `듣고 판단할 신호`를 붙인다.
- 회사 조사에서 확인되지 않은 사실을 전제로 묻지 않는다.
- 홈페이지에서 바로 답을 찾을 수 있는 질문보다 역할 기대, 팀 의사결정, 성공 기준, 제품 과제와 성장 환경을 우선한다.

## Failure Examples

- `full`인데 실무 8문항과 임원 섹션 없음: Major
- 질문마다 답변만 있고 의도·근거·꼬리질문·함정 없음: Major
- 제출 문서에 없는 A/B 테스트나 성과 수치로 답변 작성: Blocker 또는 Major
- 개인 공백 사유를 추측해 완성: Blocker
- 미경험 항목을 솔직하게 밝히고 접근 프레임을 제공: 허용
